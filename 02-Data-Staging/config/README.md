# Config Lookup Files — Provenance & Documentation

These JSON files are machine-readable lookup tables used by the data processing pipeline (Phase 2) to decode coded fields in the raw BTS TransBorder data. They were created on **2026-03-15** and validated/corrected on **2026-03-22**.

## How These Files Were Created

All code lookup files were **manually transcribed** from the official BTS data dictionary PDF and the Census Schedule D, then validated by cross-referencing against actual values found in the raw data. They were not auto-generated by script.

**Primary sources:**
- `01-Raw-Data/data_dictionary/codes-north-american-transborder-freight-raw-data.pdf` — Official BTS code reference (downloaded 2025-03-22 from bts.gov)
- `01-Raw-Data/data_dictionary/legacy-reference/codes_all.xls` — Legacy code tables bundled in BTS ZIPs
- Census Bureau Schedule D (port codes) — https://www.census.gov/foreign-trade/schedules/d/

**Additional sources for specific files:**
- `schema_mappings.json` — Derived from `01-Raw-Data/data_dictionary/Historical and current data format comparison.xlsx` (BTS-published) and `01-Raw-Data/data_dictionary/legacy-reference/Data Fields for all Tables.xls`
- `transborder_url_manifest.json` — Scraped from the BTS raw data download page at `bts.gov/topics/transborder-raw-data`

## Corrections Applied (2026-03-22)

After initial creation, all files were validated against the official BTS codes PDF. The following corrections were applied:

| File | Issue | Fix |

## Additions from Raw Data Audit (2026-03-22)

Phase 2 normalization (`03_normalize.py` step 13) audited every code value in the raw data against the config files and produced `02-Data-Staging/docs/unknown_codes_report.txt`. This revealed **73 port codes** and **4 non-port codes** present in the raw data but missing from config. The original configs were transcribed from the *current* BTS PDF and Census Schedule D, which omit retired ports, district-level catch-alls, and some historical codes.

### Port codes: 73 codes added to `schedule_d_port_codes.json`

**46 `XX` n.e.c. (not elsewhere classified) codes:**
Every customs district uses an `XX`-suffix code (e.g., `23XX`, `55XX`) as a catch-all for trade not assigned to a specific port within that district. These are a BTS convention not listed in the Census Schedule D. Port names were inferred from existing ports in the same district (e.g., `23XX` → "Laredo Customs District n.e.c." based on existing `23xx` ports being in the Laredo, TX district).

**1 code identified from the data + BTS Border Crossing dataset:**
- `2381` — Edinburg Intl Airport (Laredo, TX district). Low-volume (6 records total). Not in current Schedule D or BTS PDF.

## Known Data Quality Issues

These are documented in `schema_mappings.json` under `processing_caveats` and `known_issues`:

1. **MEXSTATE BN→BC**: Raw data from Apr 1994–May 1998 uses code `BN` (Baja California Norte) instead of the correct `BC`. The normalization script must remap this.
2. **2006 DBF field names have trailing spaces**: e.g., `'disagmot  '` — must `.strip()` field names when reading.
3. **STATMOYR format changed**: 4-char `MMYY` (1993–1997) → 6-char `YYYYMM` (1998–2006). Date parsing must handle both.
4. **2006 DOT-prefixed files** use a wider schema (254-char fields, separate MONTH/YEAR columns) that bridges to the post-2007 format.
